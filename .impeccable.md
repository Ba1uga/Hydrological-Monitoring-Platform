## Design Context

### Users
This project is used primarily in presentation and briefing contexts. The immediate feature target is the prediction dashboard's right-side chart area, which is shown during demos, 汇报, and领导/团队 walkthroughs where visual clarity matters more than dense operator controls.

### Brand Personality
The interface should feel futuristic, energetic, and authoritative. It should communicate technical confidence and live-data credibility while still feeling polished enough for big-screen presentation use.

### Aesthetic Direction
Lean into a dark large-screen dashboard aesthetic with a stronger "tech showcase" tone. Motion should be noticeable but disciplined: glowing transitions, active-state emphasis, and stage-like chart presentation instead of subtle utility tabs. The UI should avoid generic admin-card repetition and feel like a focused data theater module.

### Design Principles
1. Combine related charts into one stronger presentation surface instead of splitting attention across stacked mini-panels.
2. Make state changes obvious with clear toggle affordances and purposeful animated transitions.
3. Preserve existing data credibility by reusing current chart logic rather than inventing decorative placeholder behavior.
4. Keep the interface readable at presentation distance with strong hierarchy, large hit targets, and high-contrast active states.
5. Prefer polished motion accents over additional UI clutter.
